# Colors

Most of the components come in six variants:

- `default`: represents the base appearance of a component.

- `primary`: represents the primary call to action where we want to often direct the user's attention.

- `secondary`:

- `info`: mildly important information, a neutral color.

- `success`: useful for indicating success in alerts, can be used to indicate a contrast to `danger` below.

- `danger`: useful for indicating irreversible actions that the user may take, errors that occured when the user tries to perform an action.

- `warning`: useful for indicating a concerning matter that can be dealt with. For example, we can use it to indicate a `missing plugin` or a failed `api call` that might give you
  additional info that is not mandatory.
